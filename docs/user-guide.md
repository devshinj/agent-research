## 1. 🏠 서비스 소개

### 1.1 서비스 개요

**Crypto Paper Trader**는 업비트(Upbit) 거래소의 실시간 시세를 기반으로 한 **AI 자동매매 모의투자 플랫폼**입니다. 실제 돈이 사용되지 않으며, LightGBM 머신러닝 모델이 30개 이상의 기술적 지표를 분석하여 매매 시그널을 자동 생성합니다.

**✨ 핵심 특징:**
- 🤖 **AI 자동매매** — ML 모델이 실시간 시세를 분석하여 BUY/SELL 시그널 생성, 리스크 검증 후 자동 체결
- ✋ **수동매매 병행** — 자동매매와 별도로 거래소 페이지에서 직접 시장가/지정가 주문 가능
- 🛡️ **5단계 리스크 관리** — 서킷 브레이커, 일일 손실 한도, 포지션 제한 등 다층 안전장치
- 👥 **멀티유저 지원** — 사용자별 독립 계좌, 개인 리스크 설정, 랭킹 시스템
- ⚡ **실시간 업데이트** — WebSocket 기반 실시간 시세, 거래 알림, 포트폴리오 갱신
- 💬 **AI 분석 에이전트** — Google GenAI 기반 종목 분석 채팅

### 1.2 서비스 구조

시스템은 아래와 같은 파이프라인으로 동작합니다:

```
📡 실시간 시세 수집 → 🔍 종목 스크리닝 → 📐 기술적 지표 생성 → 🧠 ML 예측
→ 🛡️ 리스크 검증 → 📝 모의 주문 체결 → 📦 포지션 관리 → 💰 손익 정산 → 📊 대시보드 표시
```

#### 📡 데이터 수집

업비트 REST API와 WebSocket을 통해 실시간 캔들(1분/5분/일봉)과 티커 데이터를 수집합니다. 수집된 데이터는 SQLite 데이터베이스에 저장됩니다.

#### 🔍 종목 스크리닝

30분마다 전체 원화 마켓을 스캔하여 거래대금(≥10억 원)과 변동성(2~50%) 조건을 충족하는 종목을 자동 선별합니다. BTC는 항상 포함됩니다.

#### 🧠 ML 예측

LightGBM 이진 분류 모델이 약 30개의 기술적 지표(RSI, MACD, 볼린저 밴드, EMA, 거래량 비율 등)와 멀티 타임프레임(3분/10분/15분/60분/일봉) 피처를 분석합니다. 30분 후 수익률이 0.2% 이상일 확률을 예측하며, 신뢰도가 기준치 이상이면 BUY 시그널을 발생시킵니다. 모델은 1시간마다 최신 데이터로 자동 재학습됩니다.

#### 🛡️ 리스크 검증 (5단계 게이트)

모든 자동매수 주문은 아래 5단계를 모두 통과해야 체결됩니다:

| 단계 | 검증 항목 | 기본값 |
|------|----------|--------|
| ① | 🚨 서킷 브레이커 (연속 손절 한도) | 5연패 → 60분 쿨다운 |
| ② | 📉 일일 손실 한도 | 초기자금의 5% |
| ③ | 🔢 일일 거래 횟수 한도 | 500회 |
| ④ | 📦 동시 보유 포지션 수 | 최대 4개 |
| ⑤ | 💵 포지션 사이즈 + 최소 주문 금액 | 25% + 5,000원 |

#### 📝 주문 체결

PaperEngine이 슬리피지(0.05%)와 수수료(0.05%)를 적용하여 모의 체결합니다. 코인 수량은 업비트 규격에 맞게 8자리 소수점으로 절사됩니다.

#### 📦 포지션 관리

10초마다 보유 포지션을 점검하여 손절(-3%), 트레일링 스톱(고점 대비 -1.5%), 익절(+8%), 부분 익절(+4% 시 50% 매도), 추가매수(평단가 대비 -3%) 조건을 자동 실행합니다.

### 1.3 🔧 기술 스택

| 영역 | 기술 | 설명 |
|------|------|------|
| 🖥️ **백엔드** | Python 3.12+ / FastAPI | 비동기 웹 서버 + REST API |
| ⚡ **실시간 통신** | WebSocket | 시세, 거래 알림, 포트폴리오 실시간 푸시 |
| 🗄️ **데이터베이스** | SQLite (WAL 모드) | 캔들, 주문, 포지션, 시그널 저장 |
| 🧠 **ML 모델** | LightGBM | 이진 분류 (BUY/HOLD), SHAP 설명 |
| 📐 **기술적 지표** | ta (Technical Analysis) | RSI, MACD, 볼린저 밴드, EMA 등 30개+ |
| 📊 **데이터 처리** | pandas / numpy | 피처 엔지니어링, 멀티 타임프레임 집계 |
| 🔐 **인증** | JWT + bcrypt | 30분 액세스 토큰, 7일 리프레시 토큰 |
| 💬 **AI 어시스턴트** | Google GenAI SDK | 종목 분석 채팅, 웹 검색 기반 답변 |
| 🎨 **프론트엔드** | React + Vite | SPA 대시보드, 실시간 차트 |
| 💰 **금융 연산** | Decimal | 모든 금액/수량 계산에 부동소수점 대신 사용 |

### 1.4 🏗️ 아키텍처

시스템은 **6-Layer 단방향 의존성** 아키텍처를 따릅니다:

```
Layer 0  types       — 📦 도메인 모델 (Candle, Order, Position, Signal 등)
Layer 1  config      — ⚙️ YAML 설정 로더 (Pydantic 검증)
Layer 2  repository  — 🗄️ 데이터 접근 (SQLite)
Layer 3  service     — 🧠 비즈니스 로직 (ML, 매매, 리스크)
Layer 4  runtime     — 🔄 오케스트레이션 (EventBus, Scheduler)
Layer 5  ui          — 🎨 FastAPI REST + WebSocket + React 프론트엔드
```

비동기 이벤트 버스가 각 단계를 느슨하게 연결합니다:

```
📡 NewCandleEvent → 🔍 ScreenedCoinsEvent → 📶 SignalEvent → 💱 TradeEvent
```

## 2. 🚀 시작하기

### 2.1 📝 회원가입

1. 접속 주소로 이동하면 로그인 화면이 나타납니다.
2. 하단의 **"가입하기"** 링크를 클릭합니다.
3. 아래 항목을 입력합니다:
   - 📧 **이메일** - 로그인에 사용할 이메일 주소
   - 🏷️ **닉네임** - 랭킹 등에 표시될 이름
   - 🔑 **비밀번호** - 8자 이상
   - 🎟️ **초대 코드** - 관리자가 초대 코드 제도를 활성화한 경우에만 표시됩니다
4. **"가입하기"** 버튼을 누르면 계정이 생성되고, 초기 자금이 자동으로 지급됩니다.

### 2.2 🔓 로그인

1. **이메일**과 **비밀번호**를 입력합니다.
2. **"로그인"** 버튼을 클릭합니다.
3. 로그인 성공 시 대시보드로 이동합니다.

> 💡 세션은 30분간 유지되며, 만료 시 자동으로 갱신됩니다. 리프레시 토큰은 7일간 유효합니다.

## 3. 📊 대시보드

로그인 후 가장 먼저 보이는 화면입니다. 포트폴리오 현황을 한눈에 파악할 수 있습니다.

### 3.1 📈 주요 지표 (KPI 카드)

화면 상단에 4개의 핵심 지표가 카드 형태로 표시됩니다:

| 지표 | 설명 |
|------|------|
| 💰 **총 자산** | 보유 현금 + 보유 코인 평가금액의 합계 |
| 💵 **가용 현금** | 현재 매수에 사용할 수 있는 잔고 |
| 📊 **총 손익** | 전체 투자 손익 금액 (원화) |
| 📈 **수익률** | 초기 자금 대비 수익률 (%) |

총 자산 카드에는 **자동매매 상태 뱃지**가 함께 표시됩니다. KPI 카드는 WebSocket을 통해 실시간으로 갱신됩니다.

### 3.2 📉 자산 추이 차트

KPI 카드 아래에 총 자산의 변화 추이가 영역 차트로 표시됩니다.

🕐 기간 선택:
- **24시간** | **1일** | **1주** | **1개월**

> 💡 차트는 30초마다 자동으로 갱신됩니다.

### 3.3 📦 보유 포지션

현재 보유 중인 코인 목록이 테이블로 표시됩니다.

| 컬럼 | 설명 |
|------|------|
| 🪙 **종목** | 코인 이름 및 마켓 코드 (예: 비트코인 KRW-BTC) |
| 💲 **평균 매수가** | 평단가 |
| 📍 **현재가** | 실시간 시세 |
| 📊 **미실현 손익** | 평가 손익 금액 및 수익률 |
| 🔢 **수량** | 보유 수량 |
| 💵 **투자금** | 총 투입 금액 |
| 💰 **평가금** | 현재가 기준 평가 금액 |
| 🔄 **모드** | AUTO / MANUAL - 클릭하여 전환 가능 |

추가 정보:
- 🏷️ **추가매수 횟수** - 추가매수가 발생한 경우 뱃지로 표시
- 🏷️ **부분매도** - 부분 익절이 실행된 경우 뱃지로 표시

#### 🔎 포지션 상세 보기

테이블의 행을 클릭하면 해당 코인의 **캔들스틱 + 거래량 차트**가 펼쳐집니다 (최근 100봉).

#### 🔄 트레이드 모드 전환

- 🤖 **AUTO** - AI 시그널과 리스크 룰에 따라 자동으로 매매 실행
- ✋ **MANUAL** - 사용자가 직접 매매 타이밍을 결정. 수동 손절가/익절가 설정 가능

> ⚠️ 모드 뱃지를 클릭하면 확인 모달이 나타나며, AUTO로 전환하면 수동 설정된 손절가/익절가가 초기화됩니다.

### 3.4 📜 거래 내역

대시보드 하단에 최근 거래 내역이 페이지네이션(20건 단위)과 함께 표시됩니다.

| 컬럼 | 설명 |
|------|------|
| 🕐 **체결 시간** | 거래 체결 시각 |
| 🪙 **종목** | 코인 이름 및 마켓 코드 |
| 🏷️ **구분** | BUY(매수) / SELL(매도) |
| 💲 **가격** | 체결 가격 |
| 🔢 **수량** | 거래 수량 |
| 💰 **금액** | 체결 총액 |
| 📋 **사유** | ML_SIGNAL, MANUAL, STOP_LOSS, TAKE_PROFIT, TRAILING_STOP, PARTIAL_TAKE_PROFIT, ADDITIONAL_BUY 등 |

## 4. 💱 거래소 (수동 매매)

좌측 사이드바에서 **"거래소"**를 클릭하면 수동 매매 화면으로 이동합니다.

### 4.1 📋 마켓 목록

왼쪽 패널에 업비트 원화 마켓 목록이 실시간으로 표시됩니다.

**🔖 필터 탭:**
- **전체** - 모든 마켓
- **AI 스크리닝** - AI가 관심 종목으로 선정한 마켓 (AI 뱃지 표시)
- **보유** - 현재 보유 중인 종목만

**🔃 정렬:** 종목명 | 현재가 | 전일 대비 | 거래량 (오름차순/내림차순)

> 💡 가격 변동 시 **깜빡임 애니메이션**이 적용되며, WebSocket을 통해 실시간으로 갱신됩니다.

### 4.2 📈 마켓 상세 및 차트

마켓 목록에서 종목을 선택하면 오른쪽에 상세 정보가 표시됩니다:
- 🕯️ **캔들스틱 + 거래량 차트**
- 🕐 타임프레임 선택: **1분** | **5분** | **15분** | **1시간** | **4시간** | **1일**
- ⚡ WebSocket을 통해 실시간 캔들 업데이트 (새로고침 없이 최신 봉 반영)

### 4.3 🟢 매수 주문

두 가지 주문 유형을 지원합니다:

#### ⚡ 시장가 매수

1. 마켓 목록에서 매수할 종목을 선택합니다.
2. **매수 금액(KRW)**을 입력합니다.
   - 빠른 입력 버튼: 가용 현금의 **25% / 50% / 75% / 100%**
   - 최소 주문 금액: 5,000원
3. 주문 정보(예상 수량, 수수료, 슬리피지, 총 비용)가 하단에 표시됩니다.
4. **"매수"** 버튼을 클릭합니다.
5. 확인 모달에서 내용을 확인하고 **"확인"**을 누릅니다.

> 💡 시장가 매수는 현재가보다 약간 높은 가격(슬리피지 0.05%)에 체결됩니다.

#### 🎯 지정가 매수

1. 주문 유형을 **지정가**로 전환합니다.
2. **목표 가격**과 **매수 금액(KRW)**을 입력합니다.
3. **"매수"** 버튼을 클릭합니다.
4. 주문이 등록되면 해당 금액이 즉시 현금에서 차감(동결)됩니다.
5. 현재가가 목표 가격 이하로 떨어지면 자동 체결됩니다.

> ⏰ 지정가 주문은 **당일 KST 23:59:59**에 자동 만료되며, 미체결 시 동결된 금액이 환불됩니다.

등록된 지정가 주문은 매도 패널 하단에서 확인하고 취소할 수 있습니다.

### 4.4 🔴 매도 주문

보유 중인 종목을 선택하면 매도 패널이 활성화됩니다.

1. **매도 비율**을 설정합니다.
   - 빠른 입력 버튼: **25% / 50% / 75% / 전량**
2. **"매도"** 버튼을 클릭합니다.
3. 확인 모달에서 **"확인"**을 누릅니다.

> 💡 시장가 매도는 현재가보다 약간 낮은 가격(슬리피지 0.05%)에 체결됩니다. 부분 매도 후 남은 포지션 가치가 최소 주문 금액(5,000원) 미만이면 자동으로 전량 청산됩니다.

#### 🎚️ 수동 손절/익절 설정 (MANUAL 모드)

포지션이 MANUAL 모드인 경우, 매도 패널에서 **손절가**와 **익절가**를 직접 설정할 수 있습니다:
1. 원하는 가격을 입력합니다.
2. **"저장"** 버튼을 클릭합니다.
3. 현재가가 손절가 이하로 떨어지거나 익절가 이상으로 오르면 자동으로 매도됩니다.

### 4.5 💬 AI 에이전트 채팅

거래소 화면 하단에 **AI 분석 에이전트** 채팅 패널이 있습니다.

- 🗨️ 선택한 종목에 대해 AI에게 질문하거나 분석을 요청할 수 있습니다.
- 🔍 **"분석 요청"** 버튼을 누르면 해당 종목에 대한 종합 분석이 자동으로 요청됩니다.
- 📊 AI는 실시간 시세 데이터와 사용자의 현재 포지션 정보를 참고하여 답변합니다.
- ⚡ 답변은 스트리밍 방식으로 실시간 출력되며, 참고 출처 링크가 함께 제공됩니다.
- 🔄 종목을 변경하면 대화 내용이 초기화됩니다.

### 4.6 🔔 실시간 알림

거래가 체결되면 화면 우측 하단에 **토스트 알림**이 나타납니다:
- 종목명, 매수/매도 구분, 체결 가격, 사유가 표시됩니다.
- ⚡ WebSocket을 통해 즉시 전달되며, 5초 후 자동으로 사라집니다.

화면 상단 **🔔 알림 벨 아이콘**을 클릭하면 최근 50건의 거래 알림 목록을 확인할 수 있습니다. 읽지 않은 알림 수가 뱃지로 표시되며, **"모두 읽음"** 버튼으로 일괄 처리할 수 있습니다.

## 5. 🤖 전략 (AI 시그널)

좌측 사이드바에서 **"전략"**을 클릭합니다. AI 모델의 분석 결과를 확인할 수 있습니다.

### 5.1 🔍 스크리닝 결과

AI가 거래량, 변동성 등의 조건을 충족하는 종목을 자동으로 선정합니다.

📋 스크리닝 기준:
- **최소 24시간 거래대금** 충족
- **변동성** 범위 충족 (너무 낮거나 너무 높은 종목 제외)
- 스크리닝 점수 = 거래대금 x 변동성 (점수 높은 순으로 표시)

| 컬럼 | 설명 |
|------|------|
| 🪙 **종목** | 코인 이름 및 마켓 코드 |
| 💲 **현재가** | 실시간 가격 |
| 💹 **거래대금** | 24시간 거래대금 (KRW) |
| 📊 **변동성** | 최근 변동성 (%) |
| ⭐ **스코어** | AI 종합 점수 (색상으로 등급 표시) |

### 5.2 📶 트레이딩 시그널

AI 모델(LightGBM)이 생성한 매매 시그널이 시간순으로 표시됩니다.

| 시그널 | 의미 |
|--------|------|
| 🟢 **BUY** | 매수 추천 |
| 🔴 **SELL** | 매도 추천 |
| ⚪ **HOLD** | 관망 (기본 숨김, 체크박스로 표시 가능) |

각 시그널에는 **신뢰도 바**가 함께 표시됩니다. 신뢰도가 설정된 최소 기준 미만이면 HOLD로 처리됩니다.

#### 🔬 시그널 근거 확인 (SHAP 설명)

시그널에 마우스를 올리면 **SHAP 값 기반의 설명 툴팁**이 나타납니다. 상위 20개 기술 지표가 시그널에 어떻게 기여했는지 확인할 수 있습니다:

- 📊 **RSI(14), RSI(7)** - 상대강도지수
- 📈 **MACD, MACD 시그널, MACD 히스토그램** - 이동평균수렴확산
- 📏 **볼린저 밴드 상단/하단/폭** - 가격 변동 범위
- 〰️ **EMA(5/20/60) 비율** - 지수이동평균 대비 현재가 위치
- 📊 **거래량 비율(5봉/20봉), 거래량 추세** - 거래 활성도
- 📉 **수익률(1분/5분/15분/60분)** - 단기 가격 변화
- 🕯️ **고가-저가 비율, 종가 위치** - 봉 내 가격 분포

각 지표 옆에 화살표로 해당 지표가 매수/매도 방향에 기여한 정도가 표시됩니다.

### 5.3 🧠 모델 상태

각 스크리닝된 종목별 ML 모델의 현재 상태를 카드 형태로 확인할 수 있습니다:

| 항목 | 설명 |
|------|------|
| 🎯 **정확도** | 모델의 예측 정확도 (%) |
| 📚 **학습/검증 샘플** | 모델 학습에 사용된 데이터 수 |
| 🕐 **최근 학습일** | 마지막으로 모델이 학습된 시점 |
| 📊 **시그널 분포** | BUY / HOLD / SELL 비율 막대 차트 |
| 📈 **평균 신뢰도** | 시그널의 평균 확신도 |
| ⏭️ **다음 재학습** | 다음 자동 재학습까지 남은 시간 |

> 🔄 현재 재학습이 진행 중인 종목은 **"재학습 중"** 펄싱 뱃지와 경과 시간이 표시됩니다.

## 6. 🛡️ 리스크 관리

좌측 사이드바에서 **"리스크"**를 클릭합니다.

### 6.1 🚨 서킷 브레이커

화면 상단에 서킷 브레이커 상태가 큰 배너로 표시됩니다:
- 🟢 **정상** (녹색) - 자동매매가 정상 작동 중
- 🔴 **발동** (적색) - 쿨다운 카운트다운과 함께 표시

서킷 브레이커는 다음 조건 중 하나라도 해당되면 발동됩니다:
- ❌ 연속 손절 횟수가 한도에 도달
- 📉 일일 최대 손실률 초과
- 🔢 일일 최대 거래 수 초과

> ⏰ 발동 시 설정된 쿨다운 시간 동안 새로운 자동 매수가 중단됩니다. 쿨다운이 끝나면 연속 손실 카운터가 초기화되고 자동으로 해제됩니다.

### 6.2 📊 리스크 현황 지표

| 지표 | 설명 |
|------|------|
| 📉 **일일 손실률** | 오늘 누적 손실 비율 (진행 바) |
| ❌ **연속 손실** | 연속 손절 횟수 (5단계 미터) |
| 🔢 **일일 거래 수** | 오늘 체결된 총 거래 수 (진행 바) |

### 6.3 ⚙️ 개인 리스크 설정

모든 사용자가 조정할 수 있는 개인 설정입니다. 슬라이더로 조정하며, 각 항목에 마우스를 올리면 설명이 표시됩니다.

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 🔻 **손절 비율 (%)** | 매입가 대비 이 비율만큼 하락하면 자동 매도 | 3% |
| 🔺 **익절 비율 (%)** | 매입가 대비 이 비율만큼 상승하면 자동 매도 | 8% |
| 📐 **트레일링 스톱 (%)** | 고점 대비 이 비율만큼 하락하면 매도 (수익 구간에서만 작동) | 1.5% |
| 📉 **일일 최대 손실 (%)** | 하루 최대 허용 손실률 (초과 시 서킷 브레이커) | 5% |
| 📦 **포지션 비율 (%)** | 1회 매수 시 총 자산 대비 최대 투자 비율 | 25% |
| 🔢 **최대 포지션 수** | 동시에 보유 가능한 종목 수 | 4개 |

설정 변경 방법:
1. 원하는 항목의 슬라이더를 조정합니다.
2. ✅ **"적용"** 버튼을 클릭합니다.
3. 🔄 **"초기화"** 버튼을 누르면 기본값으로 되돌릴 수 있습니다.

### 6.4 🤖 자동 리스크 관리 동작

자동매매(AUTO 모드) 포지션에는 다음 리스크 룰이 자동 적용됩니다:

| 룰 | 동작 |
|----|------|
| 🔻 **손절** | 현재가 수익률이 -손절비율 이하 → 전량 매도 |
| 🔺 **익절** | 현재가 수익률이 +익절비율 이상 → 전량 매도 |
| ✂️ **부분 익절** | 수익률이 4% 이상 도달 시 50% 자동 매도 (1회만) |
| 📐 **트레일링 스톱** | 수익 구간에서 고점 대비 하락 폭이 트레일링 비율 이상 → 전량 매도 |
| ➕ **추가매수** | 매입가 대비 3% 하락 시 자동 추가매수 (최대 1회, 일반 매수의 50% 규모) |

> ⏱️ 포지션 체크는 **10초마다** 자동으로 수행됩니다.

## 8. ⚙️ 시스템

좌측 사이드바에서 **"시스템"**을 클릭합니다.

### 8.1 🤖 자동매매 제어

| 항목 | 설명 |
|------|------|
| ▶️⏹️ **자동매매 시작/중지** | 내 계정의 AI 자동매매를 켜거나 끕니다 |

> ⚠️ 자동매매를 **중지**해도 기존 보유 포지션의 손절/익절/트레일링 스톱은 계속 작동합니다. 새로운 AI 기반 매수 진입만 중단됩니다.

### 8.2 🔬 진입 분석기 (Entry Analyzer)

자동매매 시 BUY 시그널이 발생하더라도, 진입 품질 분석기가 추가 검증을 수행할 수 있습니다. 진입 분석기는 가격 위치, RSI 회복세, 추세 확인 등을 종합하여 진입 품질 점수(0~1)를 산출합니다.

관리자는 이 기능을 **실시간으로 켜거나 끌 수** 있으며(Hot Reload), 서버 재시작 없이 즉시 반영됩니다.

- ✅ **활성화 시**: 진입 품질 점수가 기준(min_entry_score) 이상인 경우에만 매수 진행
- ❌ **비활성화 시**: ML 시그널 신뢰도만으로 매수 결정

### 8.3 🔥 설정 Hot Reload (관리자)

관리자는 서버를 재시작하지 않고도 주요 설정을 실시간으로 변경할 수 있습니다. 변경 가능한 항목:

| 카테고리 | 변경 가능 항목 |
|----------|---------------|
| 🛡️ **리스크** | 손절/익절/트레일링 비율, 일일 거래 한도, 서킷 브레이커 설정, 부분 익절 비율 |
| 🤖 **전략** | 최소 신뢰도, 수익률 임계값 |
| 🔍 **스크리닝** | 최소 거래대금, 변동성 범위, 최대 종목 수, 항상 포함 종목 |
| 💱 **매매** | 포지션 비율, 최대 포지션 수, 추가매수 설정 |
| 🔬 **진입 분석** | 활성화/비활성화, 최소 점수, 가격 참조 캔들 수 |

### 8.4 🔄 계정 초기화

투자 실적을 리셋하고 처음부터 다시 시작할 수 있습니다.

1. **"계정 초기화"** 버튼을 클릭합니다.
2. 확인 모달에서 초기화를 확인합니다.
3. 모든 포지션이 청산되고, 초기 자금으로 리셋됩니다.

> ⚠️ **주의:** 계정 초기화는 되돌릴 수 없습니다. 거래 내역과 손익 기록이 모두 삭제됩니다.

## 9. ❓ 자주 묻는 질문

### Q. 실제 돈이 사용되나요?
> 💡 **아닙니다.** 이 시스템은 모의투자(Paper Trading)입니다. 실제 업비트 시세를 사용하지만, 모든 거래는 가상으로 처리됩니다.

---

### Q. 자동매매는 어떻게 동작하나요?
> 🤖 LightGBM 모델이 업비트의 실시간 시세 데이터로부터 30개 이상의 기술 지표를 분석하여 **BUY 시그널**을 생성합니다. 시그널의 신뢰도가 기준치 이상이고, 리스크 관리 조건(서킷 브레이커, 일일 손실 한도, 포지션 수 제한 등)을 모두 통과하면 자동으로 매수가 실행됩니다. 매도는 손절/익절/트레일링 스톱 룰에 의해 자동으로 실행됩니다.

---

### Q. 자동매매와 수동매매를 동시에 사용할 수 있나요?
> ✅ **네.** 거래소 페이지에서 언제든 수동으로 매수/매도할 수 있습니다. 또한 보유 포지션별로 AUTO/MANUAL 모드를 개별 설정할 수 있어, 특정 포지션만 수동 관리하는 것도 가능합니다. MANUAL 모드 포지션은 AI 매도 시그널을 무시하며, 직접 설정한 손절가/익절가에 의해서만 자동 매도됩니다.

---

### Q. 서킷 브레이커가 발동되면 어떻게 되나요?
> 🚨 연속 손절 횟수 초과, 일일 최대 손실률 초과, 일일 최대 거래 수 초과 중 하나라도 해당되면 서킷 브레이커가 발동됩니다. 기본 쿨다운 시간(60분) 동안 새로운 자동 매수가 중단됩니다. 쿨다운이 끝나면 연속 손실 카운터가 초기화되고 자동으로 해제됩니다. 수동 매매는 서킷 브레이커와 무관하게 항상 가능합니다.

---

### Q. 수수료와 슬리피지가 적용되나요?
> 💰 **네.** 매매 수수료 0.05%와 슬리피지 0.05%가 적용됩니다. 매수 시에는 현재가보다 약간 높게, 매도 시에는 약간 낮게 체결되며, 체결 금액에 대해 수수료가 부과됩니다.

---

### Q. 초기 자금은 얼마인가요?
> 💵 관리자가 설정한 금액이 자동으로 지급됩니다. 가입 후 대시보드에서 확인할 수 있습니다.

---

### Q. 지정가 주문은 언제 체결되나요?

### Q. 트레일링 스톱은 어떻게 작동하나요?
> 📐 현재가가 매입가보다 높은 수익 구간에 있을 때만 작동합니다. 보유 기간 중 최고가를 추적하며, 최고가 대비 설정한 비율만큼 하락하면 자동으로 매도합니다. 예를 들어 트레일링 스톱이 1.5%이고 최고가가 100만 원이면, 98만 5천 원 이하로 떨어질 때 매도됩니다.

---

### Q. 부분 익절이란 무엇인가요?
> ✂️ 수익률이 일정 수준(기본 4%)에 도달하면 보유 수량의 50%를 자동으로 매도하여 이익을 확정하는 기능입니다. 포지션당 1회만 실행됩니다. 나머지 50%는 익절 또는 트레일링 스톱 조건에 따라 별도로 관리됩니다.

---

### Q. 추가매수(물타기)는 어떻게 동작하나요?
> ➕ 자동매매 모드에서 보유 종목의 현재가가 평균 매입가 대비 3% 이상 하락하면 자동으로 추가 매수가 실행됩니다. 추가매수 규모는 일반 매수의 50%이며, 최대 1회로 제한됩니다. 추가매수 후 평균 매입가가 재계산됩니다.

### Q. 화면이 실시간으로 업데이트되지 않아요.
> 🔌 WebSocket 연결이 끊어졌을 수 있습니다. 사이드바에서 연결 상태 표시등을 확인하세요. 페이지를 새로고침하거나, 문제가 지속되면 다시 로그인해 보세요.

### Q. 랭킹은 어떤 기준으로 정렬되나요?
> 🏆 **수익률(%)** 기준으로 정렬됩니다. 같은 수익률인 경우 총 손익 금액이 높은 사용자가 상위에 표시됩니다.

### Q. 진입 분석기(Entry Analyzer)란 무엇인가요?
> 🔬 BUY 시그널이 발생했을 때 추가로 진입 타이밍의 품질을 검증하는 기능입니다. 최근 가격 위치, RSI 회복세, 추세 방향 등을 종합하여 0~1 사이의 점수를 산출합니다. 점수가 기준 미만이면 해당 시그널을 건너뜁니다. 관리자가 서버 재시작 없이 실시간으로 켜거나 끌 수 있습니다.

### Q. AI 에이전트 채팅은 어떤 정보를 참고하나요?
> 💬 선택한 종목의 실시간 시세(가격, 변동률, 거래대금)와 사용자의 현재 포지션 정보(보유 시 평균 매입가, 수량, 미실현 손익)를 참고합니다. 또한 Google 검색 기반의 최신 정보를 활용하여 답변하며, 참고한 출처 링크가 함께 제공됩니다.
